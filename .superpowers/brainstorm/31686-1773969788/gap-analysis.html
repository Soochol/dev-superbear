기존 페이지 vs 스펙 갭 분석

현재 prototype/index.html의 6개 페이지를 스펙과 비교한 결과입니다

1. Dashboard

갭 거의 없음

스펙에 별도 Dashboard 정의 없음. 현재 구현이 종합 요약 역할을 잘 수행.

✅ Active Cases 요약  ·  ✅ Recent Alerts  ·  ✅ Win Rate/Return 통계

2. Search (종목 검색)

소규모 갭
✅ 구현됨
  • · 자연어 검색 프롬프트
  • · 에이전트 실행 상태
  • · 프리셋 칩
  • · 결과 리스트 (Chart/Pipeline 연결)
  • · 저장된 검색 조건
❌ 빠짐
  • · 마켓플레이스 검색 조건 가져오기 UI
  • · 검색 조건 수정(편집) 모달

3. Chart (차트)

중간 갭
✅ 구현됨
  • · 차트 영역 (SVG placeholder)
  • · 타임프레임 선택 (1m~1M)
  • · MA 오버레이 라인
  • · 재무 이벤트 마커 (Earnings, Volume)
  • · Financial 사이드바 (Revenue, PER 등)
  • · AI Fusion Insight
  • · Sector Rank (간략)
❌ 빠짐
  • · PER/PBR 밴드 오버레이
  • · 매출/이익 성장률 하단 패인
  • · 기술 지표 패널 (RSI, MACD, BB 별도 차트)
  • · 섹터 비교 풀뷰 (밸류에이션+모멘텀+재무건전성)
  • · 유상증자/배당기준일 마커

4. Pipeline (파이프라인)

중간 갭
✅ 구현됨
  • · 3 섹션 구조 (분석/모니터링/판단)
  • · 블록 팔레트 (드래그 가능 형태)
  • · Stage별 순차/병렬 표현
  • · DSL 판단 스크립트
  • · 가격 알림 조건
❌ 빠짐
  • · 블록 편집 모달 (AgentBlockPrompt: objective, input_desc, tools, output_format, constraints)
  • · 자연어 → 구조화 프롬프트 변환 UI
  • · 파이프라인 목록/선택 UI
  • · 실행 상태 조회 UI (실행 중 진행률)

5. Case (케이스 관리)

큰 갭
✅ 구현됨
  • · 단일 케이스 상세 뷰
  • · 타임라인 (D-Day → 현재)
  • · 매수/매도 기록 (타임라인 내)
  • · 기본 통계 (수익률, 섹터 상관관계)
❌ 빠짐
  • · 케이스 목록 뷰 (필터: 상태/종목/섹터)
  • · 성공/실패 조건 표시 & 진행률
  • · 가격 알림 섹션
  • · 케이스 간 비교 뷰
  • · 수익률 추적 전용 섹션 (D+1, D+7, D+30...)
  • · 매수/매도 히스토리 전용 패널

6. Backtest (백테스트)

소규모 갭
✅ 구현됨
  • · 통계 요약 (이벤트, 승률, 평균 수익률)
  • · 누적 수익률 차트 (vs KOSPI)
  • · 섹터별 승률
  • · 패턴 매칭 카드
  • · AI 전략 개선 제안
  • · Best/Worst Case
❌ 빠짐
  • · 촉매 유형별 성과 (정책/실적/M&A/테마)
  • · 기간 설정 입력 UI
  • · LIVE vs BACKTEST 모드 전환

+ 완전히 새로 만들어야 할 페이지 (4개)

⬜ 모니터링 — 전체 케이스의 모니터링 현황 대시보드
⬜ 알림 — 알림 센터 (텔레그램 연동 설정 포함)
⬜ 포트폴리오 — 실현/미실현 손익, 섹터 비중, 리밸런싱
⬜ 마켓플레이스 — 파이프라인/블록/검색조건 공유 브라우저
