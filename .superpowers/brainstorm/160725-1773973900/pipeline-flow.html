핵심 변경 포인트

BEFORE (현재)

  • 파이프라인 = 범용 전략 (여러 종목 검색)
  • 종목 선택 개념 없음
  • Execute 버튼 = 1회 실행
  • 알림 출력 경로 불명확

AFTER (제안)

  • topbar에 종목 선택기 추가 (1종목)
  • Section 1 = 등록 시 1회 분석
  • Section 2 = 백엔드 주기 실행 (에이전트+DSL)
  • Section 3 = 알림 출력 (In-App/Push/Slack)
  • Register & Run 버튼으로 등록+실행

❓ 질문: 현재 노드 팔레트(왼쪽)에서 드래그해서 캔버스에 놓는 빌더 방식은 유지할까요? 아니면 위 목업처럼 섹션 기반(Analysis/Monitoring/Alert)으로 단순화할까요?
